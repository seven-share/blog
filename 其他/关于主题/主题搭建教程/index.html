主题搭建教程

步骤

  1. 首先刷一遍文档
  2. 按文档推荐的目录结构搭建自己的主题结构
    • 主题链接跳转
  3. 建议首先直接搭建网页UI，先不用hexo的框架
  4. 网页UI之后再拆开，作为hexo的一部分
  5. 本主题使用的bootstrap作为UI框架

踩坑

  1. post是文章，page是新开启一个url路径，例如分类和关于，这些page不会作为文章显示
  2. categories_dir所默认的文件夹是categories，所以新建page文件夹的时候，名字要对应好，否则分类文章路径无法进一步显示，例如这样的路径: http://localhost:4000/blog/主题/xx子分类/
  3. _post文章中可以进一步新建文件夹，进行目录结果的分类
  4. 看其他主题的源码，把头部，尾部等组件很多拆开成_partail，在开始写的时候可以不拆分，在一步步深入的情况下，逐步根据需求拆分
  5. 默认的layout是post，所以在分类等其他页面，需要在markdown里注定layout
  6. 关于路径
    • 文章路径和config.yml里的permalink有关，可以自定义
    • 进入分类的细节页面，可以直接返回该分类下的所有文章
  7. 调试过程中，可以在ejs里写<% console.log %>,查看变量

主题注意点

  1. 置顶功能，根据top值倒叙，参考1，参考2
  2. 分类页，只能是四个，在category.ejs里写死了，所以可更换或再更改模板才能添加

命令行记录

  1. hexo server 本地服务器拉起
  2. hexo g 生成文件
  3. hexo d 部署文件
