  • 读这本书，仿佛是一个经验丰富的程序员在和我对话，他的经历让我受益非其
  • 我有很多疑惑，想通过这本书找到答案

  • 很多事情，我目前还没有经历，可能只有经历过才会感受更加深刻

代码

  • 代码其实是时间和质量的平衡
  • 不断的平衡优化即可，有时还需要克制重构的冲动
  • 防御式编程，日志、监控、跟踪器(即RPC调用时的日志追踪)
  • 配置即代码，配置过于复杂时，设置默认值；将配置进行分组
  • 注意依赖管理
  • 重视测试

代码评审

  • 代码评审包括准备、提交、评审、批准合并
  • 提交评审时，描述清楚需要做的事情，可以有一个固定的模板
  • 注意提交的分支策略
