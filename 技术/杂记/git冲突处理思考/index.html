git冲突处理思考

  • 真的好久没使用git了，大概流程还记得，搜一下命令，基本就记起来了
  • 突然心血来潮，搞一下冲突处理吧，以前看过，但自己基本的还不熟，就没用过
  • 参考链接
  • 无非是git checkout,reset,revert
  • 先说git checkout，原本是为了切换分支
  • 你可以用git checkout 版本号，这样切换到某个版本
  • 做一些改动，提交。但会提示你的HEAD不在checkout的版本号那里
  • 当你准备切换回master的时候，会提示你是否将checkout的分支正式新建一个分支保存
  • 如果不新建分支，将会放弃checkout那个分支的改变
  • git reset 版本号，回滚分支
  • 此时进行commit时，该版本号之后的提交将会被删除
  • git revert 版本号，新建一个版本为该版本号
  • commit的时候，该版本号之前的版本不受影响，在其新建的版本后继续修改前进

冲突处理

  • 远程和本地有冲突的时候，git pull拉下来，此时会将远程和本地进行合并，新建一个版本
  • 然后你删除冲突，重新add，commit，push即可
  • 本地git merge的时候，如果冲突，会建一个虚拟的合并版本让你处理冲突
  • 处理完成后，提交即可
